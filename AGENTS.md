# AGENTS.md — rs-grid documentation site

Expert in TypeScript, Rspress v2, MDX, and technical documentation writing.

## Commands

```sh
npm run dev      # dev server (hot-reload)
npm run build    # production build → doc_build/
npm run preview  # preview the build locally
```

## Documentation

- rs-grid API & concepts (public, no build needed): https://rs-grid.com/llms.txt
  (index) or https://rs-grid.com/llms-full.txt (full) — French: https://rs-grid.com/fr/llms.txt
- Local build equivalents: `doc_build/llms.txt` / `doc_build/llms-full.txt`
- Any page as Markdown: append `.md` to its URL (e.g. https://rs-grid.com/getting-started.md)
- Rspress v2: https://rspress.rs/llms.txt

## Structure

```
ruxelion-site/        # repo for the rs-grid docs site (domain: rs-grid.com)
├── docs/
│   ├── en/          # English docs (source of truth)
│   │   ├── api/         — public API reference
│   │   ├── concepts/    — architecture & internals
│   │   ├── data/        — data sources & model
│   │   ├── features/    — user-facing features
│   │   ├── theming/     — CSS theme & variables
│   │   ├── integrations/— frameworks, custom-renderer, MCP
│   │   ├── scene/       — scene graph
│   │   ├── deployment/  — hosting / Docker
│   │   ├── development/ — workspace & testing
│   │   └── *.mdx        — index, getting-started, installation, demos, …
│   ├── fr/          # French docs (translation of en/)
│   └── public/      # static assets (CNAME, robots.txt, images, wasm, demos)
├── theme/           # Rspress custom theme (index.tsx)
├── doc_build/       # generated build output — never edit
└── rspress.config.ts
```

## Rules

- All pages exist in both `en/` and `fr/` — update both when editing content.
- `doc_build/` is generated by `npm run build` — never edit it by hand.
- `llms: true` is set in `rspress.config.ts` — `llms.txt` and `llms-full.txt`
  are auto-generated at build time.
- Page front matter must include `title` and `description`. Each page's
  `description` becomes its line in `llms.txt`, so write it as a standalone,
  searchable one-liner.
- The rs-grid engine repo is PRIVATE; this docs site is PUBLIC. Never reference
  private rs-grid repo paths, URLs, or internals in docs, commits, or generated
  output.
